What is Algorithmic Anaylsis Anyways

We will be learning about algorithmic analysis. Algorithmic analysis can be defined as the process of finding the computational complexity of algorithms both in time, storage, or other resources needed to execute them. For now we are first going to take a look at the time complexity aspect of an algorithm, since this is what is generally harder to discern. There is a relationship associated with this, generally between the length of the input for the algorithm and the resouces used by the algorithm. Now, of course the resouces needed for the algorithm can vary depending on the input size and type. Most obviously, the greater the size of the input the greater the amount of resouces needed. What can be less obvious is the effect the input type has on the resources required. This type can include data type of the varaibles of input structure. Lets take an example of this. Lets look at insertion sort. Insertion sort can be defined by the following psudeo-code.

algorithm Insertion Sort(List L) i = 0 while i < length(L) j = i while j > i and L[j-1] > L[j] swap L[j] and L[j-1] j = j - 1 end while i = i + 1 end while

If this pseudo-code doesn't make all that much sense, that is fine of course. However, in order to continue with the rest of this it will be beneficial to have an understanding of insertion sort. Since this is not about explaining insertion sort we will continue.

Here is an in depth explanation of insertion sort by tutorials point in case you need it though. :)

Now lets continue. Insertion sort is a great easy algorithm to look at and understand the effect the type of input has on the time complexity of the algorithm.

Complexity Theory

Best Case Time Complexity - \(\Omega\)

First, think what would happen if all the data in the input, or the type of input, was already sorted in ascending order. Emphasis on ascending. Well, based on the way insertion sort operates, the algorithm would not have to make any changes. Really, it could traverse through the whole list in one loop. Meaning the code in the main loop would only have to execute once for every item in the list, or the size of the list. The size of the list is denoted as \(n\).

This is called a best case time complexity. Why, well because this is the best case scenario for the algorithm. This will be the least amount of time an algorithm could theoretically operate in. This is best-case time complexity is denoted with \(\Omega\), a.k.a. Big-Omega.

Best-Case Time Complexity Equation

Mathematically, it is described by the following equation:

$$\Omega(g(n)) = \exists \, c > 0, n_{0} > 0$$
$$s.t. \; 0 \leq cg(n) \leq f(n)$$
$$\forall \, n \geq n _{0}$$

Lets break this down just in case you're not too familiar with the notation. Some of this is axiomatic, but I'll explain nevertheless. I will break this down into individual parts, then piece them together to arrive at the final equation. First, lets start with what's left of the equal sign.

$$\Omega(g(n))$$

\(\Omega\) is Big-Omega which denotes best-case time complexity or the lower-bounds. \(g(n)\) is describing the function of \(g\) on \(n\), a.k.a. the input size. Next, lets look at

$$\exists \, c > 0, n_{0} > 0$$

This is saying that there exists two constants, \(c\) and \(n_{0}\). \(c\) accounts for things like runtime constants such as processor speed, memory speed, etc.. It also accounts for other runtime constants in the code such as conditional statements, variable initiliazations, etc. that are asymptotically held constant. Now, these constants must be above 0.

$$0 \leq cg(n) \leq f(n)$$

Then there is \(0 \leq cg(n) \leq f(n)\). This means that \(cg(n)\) lies between \(0\) and \(f(n)\). In this instance, \(f(n)\) is the original asymptotic function, the one that is derived from the algorithm itself. \(cg(n)\) denotes the lower-bounds of our original asymptotic function. Thus, this is saying that the lower-bounds of the original asymptotic function for our algorithm lies at or below the original asymptotic function or at or above \(0\).

$$\exists \, c > 0, n_{0} > 0$$
$$s.t. \; 0 \leq cg(n) \leq f(n)$$

Now, to concatenate the last two parts together. This segment of the equation is saying that there exists two constants \(c\) and \(n_{0}\) such that our lower-bounds of the original asymptotic function \(cg(n)\) lies at or below the original asymptotic function \(f(n)\) or at or above \(0\).

So putting it all together and we have the equation:

$$\Omega(g(n)) = \exists \, c > 0, n_{0} > 0$$
$$s.t. \; 0 \leq cg(n) \leq f(n)$$
$$\forall \, n \geq n _{0}$$

From what we have learned we can say that this equation here says that there exists a constant \(c\) and \(n\), \(c\) being runtime constants, that are greater than 0 such that our lower-bound exists between 0 and the original asymptotic function.
