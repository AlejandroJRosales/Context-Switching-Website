What is a Randomized Algorithm

Randomization Algorithms are algorithms that, to some degree, implement randomization into the logic of the algorithm in order to speed up the average time complexity of the algorithm. The time complexities are improved by using randomness. We will view examples throughout this webpage, so do not worry.

How Can Randomization Help

To get an idea of randomization algorithms can help, we will look at two examples. One is more rudementary algorithm, and the other is more intricate. Both problems will follow the following format: statement of the problem, a deterministic algorithm for the solution, and how randomness can help.

Rudimentary Example

A randomization algorithm has an extra component to it, that being randomness. The randomization of the algorithm is usually made to make a choice during the execution of the algorithm. A quick example, is imagine you have a list of numbers. Such as, [0, 1, 0, 1, 0, 0]. Now, your task is to buld an algorithm that returns a boolean, or true or false response, on whether or not there exists a 1 in the list. A quick implementation would be a linear search, where you traverse the list from all the way on the left, to all the way to the right. Pretty fast algorithm, right? Well, what if the size of the list, which is usually denoted by \(n\), has 10,000,000 items inside of it instead of 6, like our example. In this case the algorithm which is normally fast, will take a lot more time to run. (A time complexity on \(O(n)\) is not a bad time complexity but is rather just an example we are using) So where does the randomization come in and how can randomization help. Well, instead of doing a linear search, you instead randomly choose an index or position in the list, and check if it is a 1 or not. If not, then check another random index to see if that position in the list is a 1. Repeat, until true. That simple. In this case, the randomization can drastically help the runtime of the algorithm. Below are examples of the two algorithms implemented in java, the nonrandomized one and the one randomized one, respectively.

program: nonrandomizedSearch.java int nonrandomizedFind(int[] randomList) { for (int num: randomList) { if (num == 1) { return true; } } return false; }

How much does this randomization help? Well we would need to compare runtime time complexities, and to do that we will need to figure out the time complexity of the algorithm. We will look at how to do this in a later section, but for now, we can agree that the algorithm with randomization is better than the linear one, in regards to average time complexity.
